# Loan Propensity CRM System
> Predicting client loan intent using behavioral data — LightGBM + SHAP + Flask

---

## Overview

This project implements a complete end-to-end system for predicting which bank clients are likely to apply for a loan within the next 30 days. It combines a machine learning model (LightGBM) with a web-based CRM interface that displays predictions, SHAP explanations, and behavioral analytics.

The system was developed as part of an internship at IdBank (Yerevan, Armenia).

## Quick Start — Run Everything from Scratch

Follow these steps in order. Each script generates the files needed by the next one.

### Step 1 — Generate client data

```bash
cd scripts
python client_generate.py
```

**Output:** `data/clients.csv` — 120 synthetic bank clients with demographics and salary.

---

### Step 2 — Generate transactions and app activity

```bash
python transactions_generate.py
```

**Output:**
- `data/transactions.csv` — ~26,000 transactions across 4 behavioral phases
- `data/application.csv` — ~46,000 mobile app sessions
- `data/client_mcc_assignments.csv` — MCC category assignments and ground truth labels

> **Note:** This script assigns `will_seek_loan` labels to clients. Exactly 50% (60 clients) are labeled as loan-seekers, the rest as non-seekers.

---

### Step 3 — Generate credit history

```bash
python credit_generate.py
```

**Output:**
- `data/credit.csv` — loan applications and approvals
- `data/payments.csv` — full payment schedules with simulated behavior (on-time, late, missed)

---

### Step 4 — Compute credit scores

```bash
python credit_score.py
```

**Output:** `data/credit_scores.csv` — credit scores (300–850) computed from 5 components: payment history, credit utilization, inquiries, DTI, and relationship data.

---

### Step 5 — Feature engineering

```bash
python feature_engineering.py
```

**Output:** `data/features.csv` — 120 clients × 78 features extracted from all data sources.

> **Important:** The observation window is strictly Jan 2024 – Sep 2025. No data from the target period (Oct–Nov 2025) is used as a feature, ensuring no data leakage.

---

### Step 6 — Train the model

```bash
python model.py
```

**Output:**
- `data/model_scores.csv` — predicted probabilities, risk tiers, and outcomes for all clients
- `data/model.pkl` — serialized LightGBM model + SHAP values + feature list
- `data/feature_importance.csv` — top features ranked by SHAP gain

> **Note:** This step computes SHAP values using `TreeExplainer`. This is required for the SHAP waterfall chart in the CRM. If SHAP is not computed here, the CRM will display a "SHAP data not available" message.

---

### Step 7 — Launch the CRM application

```bash
python crm_app.py
```

Then open your browser at:

```
http://localhost:5000
```

## CRM Interface — Three Pages

### Page 1 — Client CRM
- List of all 120 clients sorted by loan propensity
- Quick filters: All / High Risk / Top 20
- Per-client detail: recommended action, urgency level, top reasons, SHAP waterfall chart, balance timeline, transaction history, credit score

### Page 2 — Analytics Dashboard
- Business impact summary (high-probability clients, expected conversions, lift vs random)
- Portfolio-level behavioral charts: balance drop distribution, category spikes, urgency score distribution, phase transition chart

### Page 3 — Prediction & Scoring
- Scored client table with OOF probabilities and outcomes (TP/TN/FP/FN)
- ROC curve and Precision-Recall curve
- Confusion matrix
- Threshold simulator — adjust the classification threshold and see live impact on precision, recall, and F1
- Feature importance chart (top 15 by SHAP gain)
- Model summary with baseline comparison

## Model Performance

| Metric | Value |
|---|---|
| ROC-AUC (OOF) | 0.9115 |
| Average Precision (OOF) | 0.9290 |
| Accuracy | 86.7% |
| Precision (loan class) | 89.3% |
| Recall (loan class) | 83.3% |
| F1-score | 0.862 |

Evaluated using 5-fold stratified cross-validation on 120 clients (60 positive / 60 negative).

## Top Predictive Features

| Rank | Feature | Group |
|---|---|---|
| 1 | check_ratio_last_3m | App behaviour |
| 2 | balance_check_spike | Urgency score |
| 3 | spike_repair | Category spikes |
| 4 | urgency_score | Urgency score |
| 5 | failed_tx_rate | Volatility |

The dominant signal is `check_ratio_last_3m` — clients who suddenly check their balance more frequently than usual are significantly more likely to seek a loan.

## Notes

**Synthetic data:** All data in this repository is synthetically generated. No real client data is included. The model was separately validated on real bank data internally.

**Data directory:** The `data/` folder is not included in the repository and is created by running the generation scripts in order.

**SHAP requirement:** The CRM waterfall chart requires SHAP values saved inside `model.pkl`. These are computed automatically when running `model.py`. If you load an older `model.pkl` that does not contain SHAP data, re-run `model.py`.

**Port conflict:** If port 5000 is already in use, change the port in the last line of `crm_app.py`:
```python
app.run(port=5001, debug=False)
```
